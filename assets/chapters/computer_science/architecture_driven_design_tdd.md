## TDD (TEST DRIVEN DEVELOPMENT - Test-driven design/unit-testing)

The process of writing unit tests before the code they’re testing is called TDD (TEST DRIVEN DEVELOPMENT). 
I don’t care much about the output. The only thing needed is to carry out the test in a particular way.

2 groups in TDD: 

* les fondamentalistes qui exigent qu’aucune ligne de code ne doive être écrite sans un test, 
* les renieurs, ceux qui se refusent carrément à utiliser le TDD 

Projets = succès, avec et sans TDD
