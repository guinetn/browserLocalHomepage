# Gang of Four Patterns

## Creational
How the entities should be created to fit its purpose.
* Constructor
* Factory
* Abstract
* Prototype
* Singleton
* Builder

## Structural
Simple ways of relationship between different objects.
* Decorator
* Facade
* Module
* Flyweight
* Adapter and Proxy

and derived:
* Whole–Part pattern (complex objects).
* Composite pattern (nested objects and tree structures).
* Bridge pattern (allow an object to have several implementations).
* Façade pattern (creating a unified interface to a logical grouping of objects).

## Behavioral
Improving or streamlining the communication between disparate objects in a system.
* Iterator
* Mediator
* Observer
* Command and Visitor
